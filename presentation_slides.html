01 Problem Description Background
Background Context
  • Generative AI models are capable of crafting highly convincing, fabricated audio-visual content (Deepfakes).
  • Traditional classifiers suffer under complex **cross-modal** manipulations, especially visual-only or audio-only hacks.
Core Problem Statement
  • No Explainability: Existing production classifiers output a binary label (Real/Fake) without justifying the decision metrics.
  • Unseen Domain Failure: Out-of-distribution compression and generative updates cause standard networks to fail completely.
  • Lack of Localization: Models fail to specify **when** (temporal sequence blocks) and **where** (spatial face swap boundary) the edits were injected.
02 System Objectives Architectural Goals
1. Robust Joint Fusion Classifier
  • Develop a transformer-based joint representation linking facial visual cues with vocal acoustic features.
  • Enforce robustness using frozen visual/audio self-supervised weights (ViT + Wav2Vec2) to prevent overfit.
2. Multi-Channel Disentanglement
  • Disentangle anomalies into **4 independent channels**: Lip-sync mismatch, Identity swap, Temporal jumps, and AV-sync misalignment.
3. Explainable Forensic Localization
  • Apply a dilated 1D Temporal CNN + Linear-Chain Conditional Random Field (CRF) layer to predict frame-level forgery tags.
  • Construct a real-time web deployment providing visual heatmaps and dynamic temporal charts to verify findings.
03 Proposed Architecture Technical System
Pipeline Step Core Module Functional Operations / Role
1. Preprocessing RetinaFace & FFmpeg Extract raw 16kHz audio. Process visual track at target 25 FPS, isolating face crops (224x224) and mouth ROIs (96x96).
2. Feature Encoders ViT-Base & Wav2Vec2-Base Extract spatial visual embeddings (768-dim) and vocal acoustic embeddings (768-dim) from raw preprocessed inputs.
3. Cross-Modal Fusion Transformer Cross-Attention 2-layer multi-head (8 heads) cross-attention block. Dim: 512. Merges visual frame features with matching audio windows.
4. Forensic Analyzers Evidence MLP Heads Four specialist branches outputs individual anomaly scores [0, 1] (Lip Sync, Visual Identity, Temporal, and Audio-Video Sync).
5. Boundary Tagger 1D dilated CNN + Linear CRF Processes sequence anomalies across time steps, outputting CRF tags: REAL (0), FAKE (1), or BOUNDARY (2).
04 Model Hyperparameters Configuration Details
Network Dimensionality
  • Wav2Vec2 Dim: 768 (Pretrained `facebook/wav2vec2-base-960h`)
  • ViT Embed Dim: 768 (Pretrained `google/vit-base-patch16-224`)
  • Cross-Modal Dim: 512 (8 Attention Heads, 2 Cross layers)
  • CRF Boundary CNN: 3 layers, dilations=[1, 2, 4], 128 channels.
Optimization Settings
  • Optimizer: AdamW (Learning Rate: `1e-4`, Weight Decay: `1e-4`)
  • Batch Size / Accumulation: 8 / 4 gradient steps
  • Scheduler: Cosine Annealing (Max epochs: 5)

06 Observations & Contribution Innovation
Our Major Contribution
  • Unified Calibration: Calibrates overall confidence scores against decision boundaries to avoid misleadingly high-confidence predictions on edge cases.
  • Explainable UI Heatmaps: Generates frame overlays that adjust text scale dynamically depending on output resolution.
Specialist Override Logic
  • We resolved deepfake false negatives (where overall fusion probabilities are low on unseen domains) by implementing a **Specialist Override Fallback**.
  • If the classification is initially `REAL` but any individual specialist (like Visual Identity or Audio-Video Sync) outputs a confidence score exceeding **`0.60`**, the verdict is overridden to **`FAKE`**, boosting the final probability and highlighting the face region.
